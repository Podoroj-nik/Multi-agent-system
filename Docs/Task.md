# Постановка задачи

## Выбранное задание

**Мультиагентная система оценки и планирования IT-проектов**

## Суть задачи

Создать систему, которая в полуавтоматическом режиме помогает работнику анализировать заявки на IT-проекты и принимать решение об их жизнеспособности.

## Бизнес-логика

### Этап 1: Сбор требований (Скорер)
- Работник вводит описание проекта от заявителя
- Агент-скоринг задает уточняющие вопросы
- Цель: полностью понять идею проекта
- Результат: структурированное описание проекта

### Этап 2: Исследование (Серчеры)
- Три агента анализируют проект с разных сторон:
  - **Оптимист**: возможности и сильные стороны
  - **Критик**: риски и слабые места
  - **Фактолог**: рынок, конкуренты, аналоги
- Агенты могут вести дискуссию между собой
- Результат: сводный отчет с вердиктом о жизнеспособности

### Этап 3: Техническое планирование
- **Техническая группа**: план реализации MVP
- **Рекрутер**: описание вакансий для команды
- **DevOps**: рекомендации по инфраструктуре
- **Координатор**: список задач для работника

### Этап 4: Сохранение
- Отчеты сохраняются структурированно
- Доступны локально и в облаке (Яндекс.Диск)

## Роли пользователей

| Роль | Функции |
|------|---------|
| **Работник (PM)** | Вводит заявку, общается со скорером, получает отчеты |
| **Заявитель** | Предоставляет информацию о проекте через работника |

## Критерии оценки проекта системой

1. Наличие четкого описания идеи
2. Определена целевая аудитория
3. Проанализированы конкуренты
4. Оценены риски реализации
5. Сформирован план MVP
6. Определены требуемые ресурсы

## Ожидаемый результат

После прохождения всех этапов работник получает:
- Вердикт о жизнеспособности проекта
- Полный анализ с разных точек зрения
- Технический план реализации
- Список задач для запуска проекта
