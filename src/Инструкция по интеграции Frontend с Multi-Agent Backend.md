## 1. Описание API Интерфейса

Бэкенд предоставляет один основной эндпоинт для работы всех агентов.

### **POST `/process_step`**

Используется для запуска текущего агента или повторного запуска с учетом правок пользователя.

#### **Request Body (JSON):**

| **Поле**                  | **Тип**  | **Описание**                                                  |
| ------------------------- | -------- | ------------------------------------------------------------- |
| `step_index`              | `int`    | Индекс текущего агента (0–5).                                 |
| `project_description`     | `string` | Исходное описание проекта.                                    |
| `user_feedback`           | `string` | Комментарий пользователя для правок (пусто, если правок нет). |
| `previous_agents_context` | `string` | Текстовые результаты работы всех **предыдущих** агентов.      |

**Соответствие `step_index`:**

- `0`: Scorer (Скорер)
    
- `1`: Researcher (Исследователь)
    
- `2`: Admin (Администратор)
    
- `3`: HR (Команда)
    
- `4`: DevOps (Инфраструктура)
    
- `5`: Secretary (Секретарь)
    

#### **Response (JSON):**

JSON

```
{
  "result": "Текст ответа от нейросети..."
}
```

## 2. Логика взаимодействия (Frontend Flow)

Для реализации «полуавтоматического» режима ваш фронтенд должен следовать этой логике:

1. **Начало:** Пользователь вводит описание проекта.
    
2. **Шаг X (запрос):** Отправляем `POST` запрос. Если это первый запуск шага — `user_feedback` пустой.
    
3. **Отображение:** Выводим полученный `result` на экран.
    
4. **Действие пользователя:**
    
    - **Вариант А (Правка):** Пользователь пишет комментарий и нажимает «Обновить». Отправляем запрос снова с тем же `step_index`, но добавляем `user_feedback`.
        
    - **Вариант Б (Одобрение):** Пользователь нажимает «Далее». Фронтенд сохраняет `result` в локальную переменную `context`, увеличивает `step_index` на 1 и вызывает запрос для следующего агента.

## 3. Пример реализации на JavaScript (Fetch)

Вы можете использовать этот пример в любом JS-фреймворке:

JavaScript

```
async function callAgent(index, description, feedback = "", context = "") {
    const url = "http://127.0.0.1:8000/process_step";
    
    const payload = {
        step_index: index,
        project_description: description,
        user_feedback: feedback,
        previous_agents_context: context // Результаты всех шагов до этого
    };

    try {
        const response = await fetch(url, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(payload)
        });
        
        const data = await response.json();
        return data.result;
    } catch (error) {
        console.error("Ошибка при связи с сервером:", error);
    }
}
```

## 4. Важные рекомендации по хранению данных

- **Накопление контекста:** Крайне важно передавать в `previous_agents_context` все предыдущие ответы. Например, когда работает Агент-HR (`index: 3`), он должен "видеть" результаты Скорера, Исследователя и Админа, чтобы правильно подобрать команду под задачи.
    
- **Обработка JSON в ответах:** Обратите внимание, что Агент-Скорер и Агент-Админ возвращают структурированный текст (часто обернутый в Markdown блоки `json ...` ). На фронтенде может потребоваться регулярное выражение для очистки текста перед парсингом, если вы хотите отображать данные в виде таблиц.
    
- **Секреты:** Убедитесь, что файл `.gitignore` в папке `data` на бэкенде заполнен (согласно коду в `server.py`), иначе YandexGPT не получит авторизацию.
